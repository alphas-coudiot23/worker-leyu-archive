# worker-leyu-archive

本项目是一个用于归档和发布多个独立 HTML 页面的仓库。这些页面来源于不同时期、不同用途的静态内容，统一存储于此以供查阅和索引。

## 项目简介

worker-leyu-archive 旨在收集和保存各类独立的 HTML 文档。仓库本身不绑定特定域名或网站，仅作为静态页面的集中存放地。每个页面均保持原始结构，便于直接访问或二次引用。

## 目录结构

```
worker-leyu-archive/
├── index.html          # 仓库入口页面（可选）
├── pages/              # 归档页面存放目录
│   ├── page-001.html
│   ├── page-002.html
│   └── ...
└── README.md           # 本文件
```

- `pages/`：所有归档的 HTML 页面均放置于此目录下，按文件名或时间顺序组织。
- 根目录下的 `index.html` 为仓库的默认入口页（若存在），可提供导航或说明。

## 页面归档说明

- 归档的 HTML 页面均为独立文件，不依赖外部资源（除标准网络字体、样式库外）。
- 每个页面保留其原始内容与样式，不作额外修改或优化。
- 页面内容可能包含来自不同来源的静态信息，归档时已尽量保持原貌。

## 维护说明

- 本仓库接受定期更新，新增页面会按顺序加入 `pages/` 目录。
- 如需提交新页面，请确保：
  - 页面为纯 HTML 格式，无动态后端依赖。
  - 文件名具有辨识度或包含时间戳。
  - 不包含恶意代码、跟踪脚本或侵犯他人权益的内容。
- 维护者保留对归档内容进行筛选或移除的权利。

## 使用方式

1. 克隆仓库到本地：
   ```bash
   git clone https://github.com/your-username/worker-leyu-archive.git
   ```
2. 直接在浏览器中打开 `pages/` 目录下的任意 HTML 文件。
3. 也可将仓库部署到静态托管服务（如 GitHub Pages），实现在线访问。
